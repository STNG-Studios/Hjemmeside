## Claude (14:12:03)

Det afhænger af hvilken webserver/hosting du bruger:

**GitHub Pages** — Virker automatisk. Hvis der ligger en `404.html` i roden af repositoriet, bruges den automatisk for ukendte URL'er.

**Apache (de fleste webhosts)** — Kræver en `.htaccess`-fil i roden:
```
ErrorDocument 404 /404.html
```

**Nginx** — Kræver en linje i server-konfigurationen:
```
error_page 404 /404.html;
```

**Netlify / Vercel** — Bruger `404.html` automatisk (ligesom GitHub Pages).

### Sådan tester du lokalt

Start en lokal webserver og besøg en URL der ikke findes:

```bash
# Start server i projektmappen
python3 -m http.server 8000

# Besøg derefter i browseren:
# http://localhost:8000/findes-ikke
```

Bemærk: Pythons simple server viser sin egen 404 — den bruger ikke din `404.html`. For en mere realistisk test kan du bruge f.eks. `npx serve` som understøtter custom 404:

```bash
npx serve -s .
# Besøg derefter en forkert URL
```
